# Timeline Failure Taxonomy

A classification framework for identifying recurring civilizational and organizational failure modes.

Use this taxonomy when analyzing current events, historical events, market disruptions, institutional breakdowns, propaganda cycles, or system-level design flaws.

## Purpose

- standardize event analysis
- reduce vague commentary
- improve pattern recognition
- support agent prompts and automation workflows
- turn confusion into reusable lessons

## Core Categories

## 1. Bureaucratic Delay Loop
When obvious action is delayed by fragmented authority, institutional paralysis, or procedural overgrowth.

Signals:
- repeated meetings without decisive change
- slow approvals during urgent conditions
- responsibility diffusion
- policy lag behind operational reality

## 2. Propaganda Fog Cascade
When narrative overload, selective truth, emotional amplification, or information warfare obscures clear judgment.

Signals:
- contradictory messaging
- high emotion / low clarity
- memetic distortion
- public confusion about basic facts

## 3. Chokepoint Dependency Failure
When a system depends too heavily on a narrow corridor, supplier, platform, or institution.

Signals:
- shipping bottlenecks
- energy-route stress
- API single points of failure
- supply concentration risk

## 4. Empire Rerun Pattern
When old power logic repeats under new names.

Signals:
- territorial obsession
- prestige-driven escalation
- historical grievance mobilization
- inability to leave legacy competition behind

## 5. Narrative-Operational Mismatch
When slogans, public statements, or strategic messaging do not align with actual capabilities or actions.

Signals:
- official optimism during visible breakdown
- declared goals without implementation
- branding that hides structural weakness

## 6. Technological Illiteracy Spiral
When institutions or leaders make poor decisions because they do not understand the systems they depend on.

Signals:
- weak cyber hygiene
- poor data practices
- confused AI adoption
- simplistic assumptions about complex tools

## 7. Coordination Breakdown
When multiple parties or subsystems fail to synchronize.

Signals:
- duplicated effort
- conflicting directives
- missed handoffs
- integration failures

## 8. Incentive Corruption Loop
When systems reward the wrong behaviors and make failure self-reinforcing.

Signals:
- short-term gain over long-term stability
- performative metrics
- low accountability
- extractive decision cycles

## 9. Historical Memory Loss
When known lessons from prior events are ignored, forgotten, or dismissed.

Signals:
- repeated preventable mistakes
- missing after-action culture
- shallow comparisons without real learning

## 10. Complexity Without Comprehension
When systems grow too complicated for operators to understand, govern, or debug reliably.

Signals:
- fragile integrations
- unclear ownership
- unexplained failures
- maintenance debt accumulation

## Analysis Template

For each event, classify:

- primary failure mode
- secondary failure mode
- evidence observed
- consequences
- redesign proposal

## Example Record

```yaml
id: event-001
primary_failure: Chokepoint Dependency Failure
secondary_failure: Propaganda Fog Cascade
evidence:
  - supply route threatened
  - public narratives conflicting
  - leaders overstating stability
consequence:
  - cost spike
  - panic behavior
  - reduced trust
redesign:
  - diversify dependencies
  - improve transparent reporting
  - automate anomaly detection
```

## Best Practices

- prefer evidence-backed classification
- use multiple categories only when justified
- distinguish root causes from symptoms
- connect each classification to at least one redesign proposal
- keep satire separate from the base taxonomy
